# CONTRIBUTING & PEER REVIEW GUIDE — YNOR OPEN SCIENCE

## Introduction
The Ynor framework is open for external validation, criticism, and collaborative refinement. We adopt an "Open Science" approach where all theoretical foundations and spectral solvers are auditable by the international community.

## 1. Peer Review Protocol
We welcome rigorous mathematical or algorithmic reviews of the following layers:
1. **Spectral Logic:** Review of the Dirac-SUSY operator and the Riemann-Zeta correspondence.
2. **Topology:** Analysis of the chiastic symmetric structure.
3. **Information Theory:** Evaluation of the $\mu$ saturation index.

## 2. Red Teaming (Contradictory Proofs)
If a researcher identifies a logical gap or a numerical divergence ($\mu < 0.9$):
- Documentation of the divergence.
- Reproduction steps (using the provided `Dockerfile`).
- Formal submission via GitHub Issues or direct memo to the Principal Investigator.

## 3. Contribution Rules
- **Canonical Stability:** Contributions must respect the $A \to B \to C \to X \to C' \to B' \to A'$ symmetry.
- **Academic Tone:** No polemics, only formalized proofs or benchmarks.
- **License:** Contributions merge into the MDL Ynor core but retain original authorship in the `CONTRIBUTORS.md` file (to be created).
